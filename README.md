# Jenkins CI/CD Frontend — React + Vite

A React single-page application served via Nginx, deployed via Jenkins CI/CD pipeline on AWS EC2 using Docker and Ansible.

## Tech Stack

| Layer | Technology |
|---|---|
| Language | JavaScript (ES2022) |
| Framework | React 18 |
| Build Tool | Vite 5 |
| Web Server | Nginx (Alpine) |
| Containerization | Docker (multi-stage build) |
| Deployment | Ansible + Docker |
| CI/CD | Jenkins Declarative Pipeline |
| Infrastructure | AWS EC2 (provisioned by infra pipeline) |

## Pipeline Stages

```
Checkout → Read Infra State → NPM Install → NPM Build → Docker Build → Docker Push → Ansible Deploy → Smoke Test
```

| Stage | What it does |
|---|---|
| Checkout | Pulls source code from GitHub |
| Read Infra State | Reads frontend + backend IPs from `/var/lib/jenkins/tf-state/` |
| NPM Install | Installs dependencies (`npm ci`) |
| NPM Build | Builds React bundle with backend URL baked in (`VITE_BACKEND_URL`) |
| Docker Build | Builds Nginx Docker image with the static bundle |
| Docker Push | Pushes image to DockerHub |
| Ansible Deploy | SSHs into frontend EC2, pulls image, restarts container |
| Smoke Test | Hits frontend URL to confirm HTTP 200 |

## Prerequisites

- Jenkins node with: Node.js 20, Docker, Ansible
- Jenkins credentials:
  - `dockerhub-credentials` — DockerHub username + access token
  - `cicd-demo-ssh-key` — SSH private key for EC2 access
- Infra pipeline AND backend pipeline must run first

## How Backend URL is Baked In

The backend API URL is injected at build time via Vite environment variable:

```
VITE_BACKEND_URL=http://<backend-ip>:8080
```

This means the React bundle is compiled with the backend IP hardcoded. If the backend IP changes, the frontend pipeline must re-run to rebuild the bundle.

## Project Structure

```
frontend-repo/
├── Dockerfile                         # Multi-stage: Node builder + Nginx server
├── Jenkinsfile                        # Jenkins declarative pipeline
├── nginx.conf                         # Nginx config (SPA routing, caching, gzip)
├── vite.config.js                     # Vite build config
├── package.json
├── index.html
├── ansible/
│   ├── ansible.cfg                    # Ansible configuration
│   ├── inventory/hosts.ini            # Dynamic inventory (generated by Jenkins)
│   ├── group_vars/all.yml             # Default variables
│   └── playbooks/deploy-frontend.yml  # Deployment playbook
└── src/
    ├── App.jsx
    └── main.jsx
```

## Local Development

```bash
# Install dependencies
npm install

# Run dev server
npm run dev

# Build for production
VITE_BACKEND_URL=http://localhost:8080 npm run build

# Docker build
docker build \
  --build-arg VITE_BACKEND_URL=http://localhost:8080 \
  -t cicd-frontend:local .

# Docker run
docker run -p 80:80 cicd-frontend:local
```

## Ansible Deploy (what the pipeline does)

```bash
cd ansible
ansible-playbook playbooks/deploy-frontend.yml \
  -i inventory/hosts.ini \
  -e "dockerhub_username=YOUR_USERNAME" \
  -e "frontend_image_tag=latest"
```
